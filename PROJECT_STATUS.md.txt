Current focus:

* Authentication & Authorization

* Transaction Tracking
* Budget Management

# Architectural Decisions

## ADR-0002

Gateway Authentication

Decision:

Gateway validates JWT before forwarding requests.

Reason:

* Early rejection
* Reduced service load
* Centralized security boundary

Services still perform their own JWT validation.

---

## ADR-0003

Dashboard Service Cross-Service Calls

Decision:

Dashboard Service is the only service allowed to call other business services.

Reason:

* BFF aggregation use case
* Keeps core services independent
* Can be removed without impacting business services

Core services do not call each other.

---

## ADR-0004

Budget Utilization Calculation

Decision:

Budget utilization only considers categories that have active budgets.

Formula:

Budget Utilization =
Budgeted Category Spending
/
Budget Limits

Unbudgeted expenses:

* Included in total expenses
* Excluded from utilization calculations

Reason:

Avoid misleading utilization percentages.

## Error Handling

Current Policy:

Fail Fast

If any required upstream service fails:

HTTP 502

Response:

DASHBOARD_UPSTREAM_ERROR

No partial responses are returned.

Future enhancement:

Partial degradation (out of scope).

# Known Limitations

## Transaction Ordering

transactionDate is DATE-based.

Multiple transactions on the same day are ordered by:

id ASC

This is deterministic but not true creation-time ordering.

Currently acceptable.

---

## Fail-Fast Policy

Dashboard returns 502 when any required upstream service is unavailable.

This is intentional.

---

## Money Serialization

Raw monetary values may appear as:

5000.0000

Percentages may appear as:

61.43

This is a serialization-format difference, not a calculation issue.
